HTML5简介

HTML5 简介

HTML5 语法

放松的规则

  • HTML5 并不要求网页中必须包含 <html>、<head> 和 <body> 元素.
  • HTML5 不区分大小写.
  • HTML5 还允许省略关闭空元素的斜杠; 空元素就是不会嵌套内容的元素.
  • HTML5 也修改属性的语法规则. 属性值中只要不包含受限的字符(比如>、=、空格), 就可以不添加引号.
  • 只有属性名没有属性值也可以.

良好的 HTML5 风格

  • 包含可选的 <html>、<head> 和 <body> 元素. 要给页面定义自然语言, <html> 是最理想的地方; 而 <body> 和 <head> 有助于将页面内容与其他页面信息分离.

  • 标签全部小写.

  • 为属性值加引号. 防止不经意间犯错.

HTML5 验证

  • 缺少必需的元素(例如 <title> 元素);

  • 又开始标签但没有结束标签;

  • 标签嵌套错误;

  • 不包含必要属性的标签(例如没有 src 属性的 <img> 元素);

  • 元素或内容放错了地方(例如把文本直接放在了<head> 区块中).

强制使用 XHTML 语法

HTML5 元素家族

新增的元素

类别 元素
用于构建页面的语义元素 <article> <aside> <figcaption> <figure> <footer> <header> <hgroup> <nav> <section> <details> <summary>
用于表示文本的语义元素 <mark> <time> <wbr>(正式列入规范)
Web 表单及交互 <input>(增加了类型) <datalist> <keygen> <meter> <progress> <command> <menu> <output>
音频、视频及插件 <audio> <video> <source> <embed>(正式列入规范)
Canvas <canvas>
非英语支持 <bdo> <rp> <rt> <ruby>

删除的元素

<big> <center> <font> <tt> <strike> <acronym>(代之<abbr>) <aplet>(代之<object>)

改变的元素

  • <small> 表示 “附属细则”.

  • <hr> 表示主题的转换.

  • <s> 表示不在准确或不再相关的内容.

粗体斜体

​ <b> <i> 部分被 <strong> <em> 元素取代.

  • 使用 <strong> 表示 重要的文本内容, 也就是那些需要在周围文本突出出来的文本.

  • 使用 <b> 表示应该用粗体表示的文本, 但该文本并不比其他文本更重要. 比如, 关键字、产品名词等所有需要用粗体表示的文本都可以使用这个标签.

  • 使用 <em> 表示重读的文本, 也就是在朗读的时候要大声读出来.

  • 使用 <i> 表示应该用斜体表示的文本, 但该文本并不比其他文本更重要. 比如, 外文单词、技术术语等所有需要使用斜体表示的文本都可以用这个标签.

调整的元素

  • <address> 提供 HTML 文档作者的联系信息, 比如电子邮件地址或者网站链接.

  • <cite> 定义(比如书籍、歌曲、电影、电视节目、绘画、雕塑等)的标题。

  • <ol> 有了一个 reversed 属性, 用于反转序号. (还没有浏览器支持)

标准化的元素

  • <embed> 向页面中加入插件的通用方法.

  • <wbr> 在某处换行.
